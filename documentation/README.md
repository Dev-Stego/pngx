# Project Documentation

This directory contains the archival documentation for the project, including Product Requirement Documents (PRDs), implementation plans, walkthroughs, and discussion logs.

## Core Documents
- `task.md`: The master task checklist and progress tracker.
- `implementation_plan.md`: The detailed technical plan for specific features.

## Product Requirements (PRDs)
- `prd-admin.md`: Requirements for the Admin Dashboard.
- `prd-architecture.md`: System architecture overview.
- `prd-backend.md`: Backend specifications.
- `prd-database.md`: Database schema and requirements.
- `prd-frontend.md`: Frontend requirements and tech stack.
- `prd-ui-ux.md`: UI/UX guidelines and design specs.

## Plans & Walkthroughs
Various `*_plan.md` and `*_walkthrough.md` files document the step-by-step implementation and verification of specific features (e.g., Auth, UI Refinements, Security).

## Usage
Refer to these documents to understand the development history, design decisions, and future roadmap of the application.
